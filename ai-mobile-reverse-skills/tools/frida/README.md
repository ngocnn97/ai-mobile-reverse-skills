# Frida Assets

This directory contains the Frida template assets for Phase 1 / Phase 2 of the service.

The locating of these files is not "one-click kill script", but:

- Quickly generate the smallest modifiable template based on the environmental confrontation logic identified in Phase 1
- Provides a common basis for root/emulator/proxy/SSL pinning bypass in authorized environments
- Provide a reusable script skeleton for Phase 2 packet capture pre-preparation

## Current template

### android_phase1_bypass.js

effect:

- Bypass common root detection
- Bypass common emulator detection
- Block common proxies/packet capture environment detection
- Bypass common SSL Pinning / certificate verification implementation

Applicable scenarios:

- Phase 1 static reconnaissance completed
- Environmental detection that will block packet capture or runtime observation has been located.
- Requires minimal verification bypass in authorized environment

Suggestions for use:

- Prioritize combining `raw_env_guards.json`, `entrypoints.json`, `frida_bypass_plan.json` to locate the hit point before trimming the script
- Don’t treat templates as-is as being applicable to all targets
- If the target has hit the custom detection logic, the directional hook should be supplemented based on the specific class name, method name, and string.

illustrate:

- Here `android_phase1_bypass.js` is the basic template
- The actual project should be generated by Phase 1 based on the detection hits, and a customized version of the project template will be output to `{output_dir}/frida/android_phase1_bypass.js` by default.
- The generated script will indicate in the file header which modules are enabled and which hits are generated based on

Example:

```bash
frida -U -f com.example.app -l tools/frida/android_phase1_bypass.js
frida -U -n com.example.app -l tools/frida/android_phase1_bypass.js
```

## Supporting product suggestions

When Phase 1 recognizes the environmental confrontation logic, it is recommended to output at the same time:

- `raw_env_guards.json`
- `env_guard_report.json`
- `frida_bypass_plan.json`
- `frida/android_phase1_bypass.js`

in:

- `env_guard_report.json` is responsible for settling hit points, impact areas, and recommendation priorities
- `frida_bypass_plan.json` is responsible for mapping hit points to Frida hook plans
